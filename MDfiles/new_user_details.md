# Milestone 2 User Interviews — Raw Notes

## Interview Protocol

**9 total interviews across 3 iteration rounds (3 users per round).**

### Five Phases
1. **Setup** — Lower the stakes, clarify you're testing the app not the user
2. **First Glance (5-Second Test)** — Show home screen, ask what they think the app does
3. **Task (Core Loop)** — Give a scenario, observe silently
4. **AI Vibe Check** — Evaluate AI response quality, usefulness, trust
5. **Debrief** — Difficulty rating (1–5), magic wand question, one-sentence description

## Iteration 3

### Nathan

**First Glance:**
- "The user thinks that the app connects users to things and events in your area. There's like a running logo, restaurants nearby, a calendar, etc."
- "He thinks the app is a social app that is more focused on groups."
- *(Interviewer note: "this is a pretty good sign. This is very close to what our app is supposed to do and its objectives.")*

**Task:** "Imagine you are trying to make an event to gather people for Basketball. Try creating this within the app."
- "The user was able to figure out how to do this after playing around with the app for a few minutes. He mentions that most people in younger generations and most tech savvy people would be able to figure it out very quickly. It is not too different from instagram and facebook's UIs."
- "Maybe the buttons for creating new signals or missions could use more clarity, since right now, those aren't labeled or anything and are a bit unclear."
- **Bugs found:** "Profile picture is not working! Also dark mode severely screws up the user interface."

**AI Vibe Check:**
- "AI suggestions don't show up or work half the time (loading is slow). When we do get it to load, though he thinks it is pretty good and useful for helping find events that match with user interests."
- "The suggestion isn't super strong since the profile is limited in what it has. However, having it is better than not having this."
- "User would use this information in order to find events that he enjoys more, and have an easier time finding events."
- "He would use this info mainly for event finding. (The feature does not do people matching). This is moderately useful."

**Debrief:**
- Difficulty: 2.5
- Magic wand: "make UI the same for light and dark mode (since dark mode messes up the UI)"
- "Discovery page should not have templates. It should just have events available (since more people likely would want to join events than create events)."
- One sentence: "A good event finder app for finding and hosting local events to make friends at."
